How Ionic Builder Works

{% endblock %} {% block content %}

Ionic Builder is pretty bare bones at the moment. It's really just a proof of concept that happens to be somewhat useful. In its current form, we make a number of assumptions about your project (eg, it's in a public GitHub repo and you want builds done when you push to master).

To get started, all we need is the GitHub URL of your project. You can elect to have a build created right away, or you can wait until you do your next push to master. Then, you need to head over to your GitHub repo and create a webhook to notify Builder when you've pushed new code and it's time to build.

That's pretty much all you need to do. Going forward, when you push new code to master GitHub will notify us via the webhook you created. When we get that hook, we do the following:

  • Create a new Docker build container for your project.
  • Clone your repo into that container.
  • Add the Android build target to your project.
  • Execute the Ionic build command.
  • Create an archive of the APK's that were just built.
  • Put that archive in a public download location.
  • Finally, delete the Docker container.

Each time your project is built, any prior build archives will be overwritten by the new one. Your builds always happen in a fresh build container, so there's no extra crap laying around from any other builds.
